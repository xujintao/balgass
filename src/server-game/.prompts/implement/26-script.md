# 脚本系统实现提示词

你现在要为 Go 项目 `server-game` 实现并补强“脚本系统”。

## 背景

你现在位于 Go 项目 `server-game` 的仓库根目录：

/home/pi/balgass/src/server-game

另有 C++ 参考项目 `GameServer`：

/home/pi/balgass-igc/igc/9.5.1.15/source/GameServer

`server-game` 参考 C++ `GameServer` 重构而来。本模块在总表中的定位是：Lua 运行时、WZ/SMD 解析、脚本调用、Go/Lua 绑定、脚本化 Bag、任务/副本/Buff 脚本基础设施。

本模块覆盖脚本基础设施：Lua VM 生命周期、Lua 文件加载、Lua 函数调用、Go/Lua 绑定、WZ/SMD 文本脚本解析、脚本错误处理、脚本热重载、自检和测试工具。脚本系统不拥有公式、掉落、任务、Buff、副本、活动、怪物 AI 等业务语义；这些模块只调用脚本系统执行脚本或解析配置。GameServer 中 `MuLua`、`LuaFun`、`LuaExport`、`LuaBag`、`QuestExpLuaBind`、`DoppelgangerLua`、`BuffScriptLoader`、`ReadScript/WzMemScript` 均存在真实调用链，server-game 当前只有公式包私有 Lua wrapper，尚无通用脚本系统。

本提示词来自 `.prompts/todo/26-script.md`，但不是要求一次实现全部 TODO。你需要把 TODO 转换成一次可闭环、可测试、风险可控的第一阶段开发任务。

## 模块边界

本模块只处理“脚本系统”第一阶段所需的核心能力。跨系统能力只能调用对应系统或预留接口，不要顺手实现其他大模块。

本模块涉及的主要子模块分类包括：模块边界与总入口、Lua 运行时、Lua 调用封装、Lua 绑定与导出、公式脚本接入、Bag/LuaBag 接入、QuestExp 脚本接入、DoppelGanger 脚本接入。

## 第一阶段目标

先实现 Lua/WZ/SMD 解析与脚本调用基础设施，不做具体业务脚本。

第一阶段必须能独立验收。不要试图一次性实现 `.prompts/todo/26-script.md` 中所有条目。

## 实现要求

- 不修改 `handle` 层协议表，除非第一阶段目标明确需要接入已存在协议且没有其他入口。
- 不跨模块实现完整业务；需要其他模块能力时，优先调用已有接口或定义窄接口。
- 保持当前项目风格，避免引入大型框架或全局 service locator。
- 对状态变化、事务、回滚、并发访问和失败路径给出清晰处理。
- 对尚未支持的复杂功能保留明确扩展点，不要静默吞掉错误。
- 直接修改代码时，必须补充或更新相应测试；如果无法测试，要说明原因。
