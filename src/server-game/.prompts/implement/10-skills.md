# 技能系统实现提示词

你现在要为 Go 项目 `server-game` 实现并补强“技能系统”。

## 背景

你现在位于 Go 项目 `server-game` 的仓库根目录：

/home/pi/balgass/src/server-game

另有 C++ 参考项目 `GameServer`：

/home/pi/balgass-igc/igc/9.5.1.15/source/GameServer

`server-game` 参考 C++ `GameServer` 重构而来。本模块在总表中的定位是：普通技能、大师技能、技能学习、技能消耗、技能范围、技能 CD、Combo。

本模块覆盖普通技能、大师技能、技能学习、技能列表、技能使用、资源消耗、目标校验、范围命中、冷却防挂、Buff/状态、Combo、协议与测试。战斗伤害主体归对象系统和公式系统，本模块只记录技能选择、技能规则、技能效果入口和协议边界。

本提示词来自 `.prompts/todo/10-skills.md`，但不是要求一次实现全部 TODO。你需要把 TODO 转换成一次可闭环、可测试、风险可控的第一阶段开发任务。

## 模块边界

本模块只处理“技能系统”第一阶段所需的核心能力。跨系统能力只能调用对应系统或预留接口，不要顺手实现其他大模块。

本模块涉及的主要子模块分类包括：技能表与基础定义、技能实例与持久化、技能学习与遗忘、技能列表与协议、技能使用入口与消耗、目标校验与安全限制、范围与命中形状、冷却、防挂与使用频率。

## 第一阶段目标

补强技能学习/使用校验、技能列表协议、基础消耗和冷却。

第一阶段必须能独立验收。不要试图一次性实现 `.prompts/todo/10-skills.md` 中所有条目。

## 实现要求

- 不修改 `handle` 层协议表，除非第一阶段目标明确需要接入已存在协议且没有其他入口。
- 不跨模块实现完整业务；需要其他模块能力时，优先调用已有接口或定义窄接口。
- 保持当前项目风格，避免引入大型框架或全局 service locator。
- 对状态变化、事务、回滚、并发访问和失败路径给出清晰处理。
- 对尚未支持的复杂功能保留明确扩展点，不要静默吞掉错误。
- 直接修改代码时，必须补充或更新相应测试；如果无法测试，要说明原因。

## 推荐实现顺序

1. 阅读 `.prompts/todo/10-skills.md`，按状态筛出 `需修正`、`部分覆盖` 和第一阶段必须的 `未覆盖` 条目。
2. 阅读上面列出的 server-game 文件，确认当前入口、数据结构、协议结构和已有测试。
3. 阅读上面列出的 GameServer 文件，提炼业务语义、状态机、校验点和失败路径。
4. 设计最小 Go 接口和数据流，避免把其他模块的完整规则写进本模块。
5. 先修复 `需修正` 项，再补强 `部分覆盖` 项，最后实现第一阶段必要的 `未覆盖` 项。
6. 接入现有调用点，确保重复调用、断线、失败、回滚或关闭流程安全。
7. 增加单元测试或小范围集成测试，覆盖成功路径和关键失败路径。
8. 运行相关测试，最后总结已完成内容、保留内容和风险。
