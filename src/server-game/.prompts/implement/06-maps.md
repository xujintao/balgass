# 地图系统实现提示词

你现在要为 Go 项目 `server-game` 实现并补强“地图系统”。

## 背景

`server-game` 参考 C++ `GameServer` 重构而来。本模块在总表中的定位是：地图数据、地图属性、阻挡、安全区、Gate、MoveCommand、小地图、地图对象容器。

本模块覆盖地图数据、地形属性、站位占用、随机坐标、路径与墙体、Gate、MoveCommand、小地图、地图物品、天气/经验/Zen、跨服地图归属与地图侧出生/刷怪点位。对象移动、视野、攻击、死亡归对象系统；怪物 AI 主体归 `25-monster-ai.md`；移动作弊、穿墙审计、异常坐标处罚归 `27-security.md`；跨服迁移请求、地图服认证和外部服状态归 `28-external-comm.md`；本模块只提供地图规则和路径能力。

本提示词来自 `.prompts/todo/06-maps.md`，但不是要求一次实现全部 TODO。你需要把 TODO 转换成一次可闭环、可测试、风险可控的第一阶段开发任务。

## 请先阅读 server-game 文件

- game/maps
- game/object/object.go
- game/object/move.go
- conf/config.go

同时阅读 `.prompts/todo/06-maps.md`，重点关注 `需修正`、`部分覆盖`、`未覆盖` 状态的条目。

## 请参考 GameServer 文件

- MapClass
- MapServerManager
- Gate
- MoveCommand
- MapAttribute

参考 C++ 的业务语义和边界，不要照搬 C++ 的全局对象、线程模型和数据结构。Go 版应优先遵守当前 server-game 的包结构、对象协程/主循环模型和已有协议模型。

## 模块边界

本模块只处理“地图系统”第一阶段所需的核心能力。跨系统能力只能调用对应系统或预留接口，不要顺手实现其他大模块。

本模块涉及的主要子模块分类包括：地图注册与加载、地形属性文件、地图属性规则、站位占用、随机坐标与出生点、路径与墙体、对象地图移动入口、Gate 传送。

## 第一阶段目标

补强地图属性、阻挡、安全区、Gate、MoveCommand 和地图对象容器。

第一阶段必须能独立验收。不要试图一次性实现 `.prompts/todo/06-maps.md` 中所有条目。

## 实现要求

- 不修改 `handle` 层协议表，除非第一阶段目标明确需要接入已存在协议且没有其他入口。
- 不跨模块实现完整业务；需要其他模块能力时，优先调用已有接口或定义窄接口。
- 保持当前项目风格，避免引入大型框架或全局 service locator。
- 对状态变化、事务、回滚、并发访问和失败路径给出清晰处理。
- 对尚未支持的复杂功能保留明确扩展点，不要静默吞掉错误。
- 直接修改代码时，必须补充或更新相应测试；如果无法测试，要说明原因。

## 推荐实现顺序

1. 阅读 `.prompts/todo/06-maps.md`，按状态筛出 `需修正`、`部分覆盖` 和第一阶段必须的 `未覆盖` 条目。
2. 阅读上面列出的 server-game 文件，确认当前入口、数据结构、协议结构和已有测试。
3. 阅读上面列出的 GameServer 文件，提炼业务语义、状态机、校验点和失败路径。
4. 设计最小 Go 接口和数据流，避免把其他模块的完整规则写进本模块。
5. 先修复 `需修正` 项，再补强 `部分覆盖` 项，最后实现第一阶段必要的 `未覆盖` 项。
6. 接入现有调用点，确保重复调用、断线、失败、回滚或关闭流程安全。
7. 增加单元测试或小范围集成测试，覆盖成功路径和关键失败路径。
8. 运行相关测试，最后总结已完成内容、保留内容和风险。

## 重点检查并修复的问题

- 对象地图移动入口 / 移动中阻挡中断：这里使用 `&&` 可疑，通常任一阻挡属性就应中断。（当前状态：需修正）
- 对象地图移动入口 / 起点坐标同步：当前过度信任客户端起点，需限制与服务端当前位置距离。（当前状态：需修正）
- 地图物品与 Zen / 地图物品坐标校验：应过滤阻挡/站位/安全区限制，而不是只检查越界。（当前状态：需修正）
- 地图注册与加载 / 最大地图数量边界：需要统一所有入口的 map number 越界检查，避免直接索引 panic。（当前状态：部分覆盖）
- 地图注册与加载 / 重复地图名处理：重名地图如多层地图要有稳定命名规则，避免调试订阅混乱。（当前状态：部分覆盖）
- 地形属性文件 / `.att` 文件头读取：当前未显式校验文件长度和 head，需补坏文件测试。（当前状态：部分覆盖）
- 地形属性文件 / 阻挡点缓存：主要用于 Web 调试地图，需明确不参与正式逻辑。（当前状态：部分覆盖）
- 地形属性文件 / 坐标转索引：Go 侧用 `m.width`，GameServer 多处固定 256；需确认非 256 地图兼容。（当前状态：部分覆盖）
